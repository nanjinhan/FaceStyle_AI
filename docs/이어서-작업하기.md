## 1. 이 앱이 뭔가

**친구들이 단체사진 한 장에 동시 접속해서 각자 자기 얼굴만 보정하는 앱.**

기존에는 사진을 보정 → 저장 → 카톡 전송 → 친구가 또 보정 → 저장을 반복하느라
JPEG 재압축이 쌓여 화질이 계속 나빠졌다. 이 앱은 **픽셀을 주고받지 않는다.**
서버에는 원본 1장만 있고, 오가는 건 "밝기 +12, 턱 -5" 같은 파라미터 JSON뿐이다.
최종 저장할 때 원본 해상도에 딱 한 번 렌더링하므로 **몇 번을 고쳐도 화질 손실이 0**이다.

- 상세 기능: [기능명세서.md](기능명세서.md)
- 설계·프로토콜: [아키텍처.md](아키텍처.md)
- 마일스톤 전체: [로드맵.md](로드맵.md)
- 출시 준비물(스토어·키·비용): [출시-준비물.md](출시-준비물.md)

**확정된 방향**: 단체사진 전용. 얼굴 1개당 1명 클레임, 본인 영역만 편집.
셀카(얼굴 1개) 여럿이 함께 보정하는 시나리오는 지원하지 않는다.

## 2. 지금 상태 한눈에

| | 상태 |
|---|---|
| 백엔드 (FastAPI) | 실시간 방 기능 **거의 완성**. 앨범·알림·회원관리는 미착수 |
| 앱 (Flutter) | 실시간 방/에디터만 동작. 로그인·홈·앨범·알림·카메라는 **빈 껍데기 화면** |
| 테스트 | 백엔드 57개, 앱 23개 — 전부 통과 |
| 실기기 검증 | ❌ **아직 못 함** (아래 4장 참고) |

### 실제로 되는 것

- 방 만들기(사진 여러 장), 초대 링크/코드로 참여, 게스트 참여
- 여러 명 동시 접속 → 한 사람이 슬라이더 움직이면 **다른 사람 화면에 즉시 반영**
- 슬라이더 6종(밝기/대비/채도/색온도/하이라이트/그림자)이 **사진에 실제로 적용됨**
- 내 얼굴만 편집 가능, 남의 얼굴 건드리면 서버가 거부 + 이유 안내
- 실행취소가 **내 작업만** 되돌림 (친구 작업 안 건드림)
- 항목별 리셋, 전원 완료 시 자동 확정 + 편집 잠금

### 아직 안 되는 것

- 로그인/회원가입 (지금은 로그인 화면에서 그냥 통과)
- 홈 화면, 앨범, 알림, 인앱 카메라 — 전부 TODO 주석만 있는 껍데기
- **얼굴 클레임 UI** — 서버는 준비됐는데 앱에 "이게 나예요" 버튼이 없음
- 실시간 커서 표시 — 서버는 좌표를 전달하는데 앱이 안 그림
- 얼굴 워핑·피부 보정 (지금은 전역 색보정만)
- 갤러리 저장

## 4. 지금 막혀 있는 것 (작업 전 결정 필요)

### ① 얼굴 좌표를 만들 방법이 없다 — **4단계 진행 전 필수**

사진을 올려도 `faces`가 항상 빈 배열이라 **클레임할 얼굴이 없다.**
얼굴 검출 경로가 [face_detection.py](../backend/app/face_detection.py) 서버 OpenCV 하나뿐인데,
`opencv-python-headless`가 [requirements.txt](../backend/requirements.txt)에 주석 처리돼 있다.
(지금까지 테스트는 DB에 얼굴 행을 직접 넣어 우회했다.)

한편 [아키텍처.md](아키텍처.md)는 "온디바이스 MLKit이 기본, 업로드 시 bbox를 서버에 등록"이라고
적어놨는데 **그 등록 API가 존재하지 않는다.** 문서와 코드가 어긋난 지점.

- **(a)** opencv 주석 해제 — 30초. 정확도 낮지만 개발용으로 충분
- **(b)** `POST .../faces` bbox 등록 API + 앱에 MLKit 연동 — 정석, 1.5일
- **(c)** (a)로 지금 뚫고 MLKit은 M4로 ← **추천**

### ② 실기기 테스트 환경이 없다

Android SDK 미설치라 에뮬레이터를 못 쓴다. 그래서 **"두 기기에서 실제로 같이 보정된다"를
아직 눈으로 확인하지 못했다.** (렌더링 수식은 PNG 뽑아서 검증 완료 → `app/build/filter_preview.png`)

- **(a)** Android Studio 설치 (~10GB)
- **(b)** 웹 빌드 추가(`flutter create --platforms=web .`) → 크롬 창 2개로 테스트.
  백엔드 CORS는 이미 열려 있어 바로 됨. **빠르게 확인만 할 거면 이쪽**

### ③ Flutter 버전 불일치

이 PC는 Flutter 3.41.9(Dart 3.11.5)인데 프로젝트는 원래 Dart ^3.12.2를 요구했다.
진행을 위해 `pubspec.yaml`을 `^3.11.0`으로 낮춘 상태(`5f072a2`).
팀에서 Flutter 버전을 맞추고 되돌리는 게 맞다.

### ④ 백엔드 스키마 변경 — 기존 로컬 DB 폐기 필요

`EditSession.global_edit_policy`, `Photo.finalized_at`, `PhotoCompletion` 테이블이 추가됐다.
SQLAlchemy `create_all`은 기존 테이블에 컬럼을 못 붙이므로,
**예전에 서버를 돌린 적 있으면 `backend/togethersnap.db`를 지우고 다시 켜야 한다.**
또 `POST /sessions`의 필드명이 `file` → `files`로 바뀌었다.

## 6. 개발 환경 세팅

```bash
# 백엔드
cd backend
python -m venv .venv
.venv/Scripts/python.exe -m pip install -r requirements.txt   # Windows
.venv/Scripts/python.exe -m pip install pytest httpx           # 테스트용
.venv/Scripts/python.exe -m uvicorn app.main:app --reload      # http://localhost:8000
.venv/Scripts/python.exe -m pytest tests/ -q                   # 57개

# 앱
cd app
flutter pub get
flutter analyze
flutter test                                                   # 23개
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:8000    # 안드로이드 에뮬레이터
```

API 주소 기본값은 `http://10.0.2.2:8000`(안드로이드 에뮬레이터에서 본 호스트 PC).
실기기면 PC의 LAN IP를 `--dart-define`으로 넘긴다.

새 대화창에서 아래를 그대로 붙여넣으면 된다.

```
FaceStyle 프로젝트를 이어서 작업할 거야.
docs/이어서-작업하기.md 를 먼저 읽고, 이어서 docs/기능명세서.md,
docs/아키텍처.md, docs/로드맵.md 를 확인해줘.

그 다음 실제 코드와 문서가 일치하는지 확인하고,
- 지금 어디까지 돼 있는지
- 다음에 뭘 해야 하는지
- 시작 전에 내가 정해줘야 할 게 있는지
정리해서 알려줘. 그리고 진행하자.
```

특정 작업만 시키고 싶으면 뒤에 덧붙인다:

```
이번엔 M1 4단계(얼굴 클레임 UI)만 하자.
docs/이어서-작업하기.md 4장 ①번 얼굴 좌표 문제는 (c)안으로 간다.
```

### 작업할 때 지켜온 규칙

- **명세서가 기준.** 코드와 명세가 다르면 명세를 따르되, 어긋난 지점은 문서에 기록한다
- API/프로토콜을 바꾸면 [아키텍처.md](아키텍처.md)에 **먼저** 반영하고 구현한다
- 권한·동시편집처럼 조용히 깨지는 기능은 **반드시 테스트를 붙인다**
  (특히 "남의 얼굴 못 건드림" 같은 건 공격 시나리오로 검증)
- 작업이 끝나면 이 문서의 2·3·5장과 헤더 날짜/커밋을 갱신하고 커밋한다
